### 流程

```
茶妹頻道 → [12] girl_scraper → CSV + 圖片 → 上架網站
```

### 資料來源（已加入的頻道）

| 頻道 | 內容 |
|------|------|
| 精選妹子速遞-渴渴外送茶坊 | 妹子班表、照片 |

| 渴渴學生專區 | 學生類型 |
| 人妻少婦重口味專區 | 特定類型 |

### 抓取內容

每則訊息會產出：
- **文字**：妹子介紹（名字、身高體重、服務內容、價格）
- **圖片**：妹子照片（下載到本地）
- **來源連結**：`t.me/xxx/123` 直接連結
- **頻道連結**：來源頻道
- **時間**：發布日期
- **觀看數**：熱門程度參考

### 輸出格式

```
data/
├── images/
│   ├── 精選妹子速遞/
│   │   ├── 101.jpg
│   │   ├── 102.jpg
│   │   └── ...
│   ├── 粉粉嫩妹區/
│   └── ...
├── girls_精選妹子速遞_20260402.csv
└── girls_粉粉嫩妹區_20260402.csv
```

### 上架網站流程

1. 跑 `[12] 妹子資料爬取` 下載所有頻道的妹子資料
2. 整理 CSV（過濾重複、無圖的）
3. 圖片+文字批次匯入網站
4. 定期重跑更新（新妹子上架）

### 執行指令

```bash
python3 tg.py  # 選 [12]
# 選擇茶妹頻道 → 設定數量 → 開始
```

## 每日執行 SOP

### 早上（帳號 1）

1. `python3 tg.py` → `[2] 批次爬取` — 抓新加入群組的成員
2. `python3 tg.py` → `[12] 妹子資料爬取` — 下載茶妹頻道的新圖片
3. 休息 10 分鐘
4. `python3 tg.py` → `[5] 連結探索` 或 `[10] 深度爬蟲` — 發現新群組
5. 如果被限速 → 切帳號 2

### 下午（帳號 2 或帳號 1 恢復後）

6. `python3 tg.py` → `[3] 大量搜尋` — 用關鍵字找新群組
7. `python3 tg.py` → `[8] 自動私訊` — 開始導流（每日 30 則/帳號）

### 晚上

8. `python3 tg.py` → `[7] 合併去重` — 確認今日成果
9. 妹子資料整理上架網站
