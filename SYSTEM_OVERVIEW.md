## 完整流程一句話

撈名單 → 抓圖文發到自己頻道（替換連結）→ 多帳號輪換私訊導流 → Bot 承接客服

## 設備需求

| 項目 | 規格 |
|------|------|
| 電腦 | 1 台 Mac（你目前的即可） |
| TG 帳號 | 10 個（各自獨立電話+API_ID/HASH） |
| 代理 IP | 選配（不買也能跑，錯開時間降低風險） |
| Bot | 1 個（@teaprincess_bot，已建好） |


## 多帳號運作模式

```
一台電腦，一個終端機，一行指令啟動：

08:00  帳號1 啟動 → 發 25 人（約 50 分鐘）→ 斷線
08:50  休息 30 分鐘
09:20  帳號2 啟動 → 發 30 人（約 60 分鐘）→ 斷線
10:20  休息 30 分鐘
10:50  帳號3 啟動 → 發 25 人 → 斷線
 ...
18:00  帳號10 完成

每日總發送量：約 250-300 人
全程自動，無需手動切換
```


## 防封號策略

| 策略 | 做法 |
|------|------|
| 帳號隔離 | 每帳號獨立 API_ID/HASH |
| 時間錯開 | 一次只跑一個帳號，中間休息 30 分鐘 |
| 擬人化 | 模擬打字、分段發送、隨機延遲 |
| 話術輪換 | 5 組模板隨機，每人收到不同內容 |
| 延遲差異化 | 每帳號設不同間隔（60-180 秒不等） |
| 每日上限 | 新號 15-20 人/天，老號 30-40 人/天 |
| 限速偵測 | FloodWait 自動等待，PeerFlood 自動停用帳號 |
| 斷點續傳 | 中斷重跑不重複，進度自動保存 |


## 防重複發送機制

```
第 1 層：名單預分配
  → 開始前就把名單切好，每帳號拿到完全不同的人

第 2 層：即時記錄
  → 每發完一人立刻寫入 dm_sent_log.csv

第 3 層：啟動檢查
  → 每次啟動先讀取記錄，跳過所有已發送的人
```


## 日常操作 SOP

### 每日一次
```bash
# 1. 啟動多帳號私訊（全自動，跑完自己停）
cd ~/Downloads/tg-scraper-toolkit-main
source venv/bin/activate
PYTHONPATH=. python3 3_dm_multi.py
```

### 需要時執行
```bash
# 撈新名單
PYTHONPATH=. python3 1_scraper.py

# 重發內容到頻道
PYTHONPATH=. python3 2_forwarder.py

# 管理 Bot 客服
PYTHONPATH=. python3 5_bot.py
```

### Bot 保持運行
```bash
# 背景運行 Bot（關終端機也不會停）
nohup env PYTHONPATH=. python3 5_bot.py > bot.log 2>&1 &
```
