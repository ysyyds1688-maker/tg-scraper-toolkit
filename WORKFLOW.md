## 帳號分配（10 個帳號 + 1 個主帳號）

### 總覽

| 角色 | 帳號 | 任務 | 風險等級 |
|------|------|------|---------|
| 主帳號 | 茶王公主殿下 | 管理 Bot、手動觀察、備用 | 不碰自動化，永遠安全 |
| 探路號 | 帳號 1-2 | 搜尋+加入新群組、撈名單 | 高（封了直接換新的） |
| 內容號 | 帳號 3 | 即時監聽轉發到頻道 | 低（不是私訊，不算騷擾） |
| 私訊號 | 帳號 4-10 | 每天輪換發私訊導流 | 中（封一支還有六支） |

### 為什麼這樣分

- **主帳號不跑自動化** — 這是門面，絕對不能被封
- **2 支探路號** — 加群+撈名單風險最高，封了換新的就好，不影響其他功能
- **1 支內容號** — 轉發到自己頻道風險低，1 支就夠，24 小時跑
- **7 支私訊號** — 私訊是核心產能，越多越好，封一支還有六支繼續跑

### 封號應對

```
私訊號被封 1 支 → 還有 6 支繼續跑，產能掉 15%
私訊號被封 2 支 → 買 2 張新預付卡補上
探路號被封     → 直接換新的，不影響私訊和內容
內容號被封     → 把一支私訊號改成內容號
主帳號         → 永遠安全，不碰自動化
```

## Phase 1：撈名單（探路號 1-2）

```
自動化流程：

1. 關鍵字搜尋新群組
   └── 用預設關鍵字搜尋 → 找到新群組

2. 自動加入（帶冷卻）
   ├── 每加一個群等 30-60 秒
   ├── 一次最多加 8-10 個群就停
   ├── 休息 2-3 小時再加下一批
   ├── 每天最多加 15-20 個群
   └── 被限速 → 自動等待

3. 撈取成員
   └── 新加入的群組自動撈成員

4. 合併去重
   └── 產出 all_members.csv

排程：每週自動執行一次
狀態：✅ 已完成基礎功能，待整合自動加群+冷卻機制
```

## 防封號策略

| 策略 | 做法 |
|------|------|
| 帳號分工 | 主帳號不碰自動化、探路號做高風險、私訊號專門發 |
| 獨立 API | 每帳號各自的 api_id/api_hash |
| 獨立電話 | 每帳號不同手機號碼 |
| 時間錯開 | 一次只跑一個帳號，中間休息 30 分鐘 |
| 擬人化 | 模擬打字、分段發送、隨機延遲 60-180 秒 |
| 話術輪換 | 5 組模板隨機，每人收到不同內容 |
| 每日上限 | 新號 15-20 人，老號 30-40 人 |
| 加群冷卻 | 每次加 8-10 個群，休息 2-3 小時 |
| 限速偵測 | FloodWait 自動等待，PeerFlood 自動停用帳號 |
| 斷點續傳 | 中斷重跑不重複，進度自動保存 |

## 名單可發送範圍

不只有 username 的能發，**有 user_id 也能發私訊**。發送優先順序：
1. 有 username → 用 username 發
2. 沒 username 但有 user_id → 用 user_id 發

所以 all_members.csv 裡的人**全部都能發**，不只是 members_with_username.csv 那些。

---

## 防重複發送（三層保險）

```
第 1 層：名單預分配 → 每帳號拿到完全不同的人
第 2 層：即時記錄 → 每發一人立刻寫入 dm_sent_log.csv
第 3 層：啟動檢查 → 重跑自動跳過已發的人
```
